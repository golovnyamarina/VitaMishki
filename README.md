**ВитаМишки** — это современный интернет-магазин специализированных детских витаминов. Проект спроектирован с глубоким пониманием B2C-специфики ниши: строгой возрастной сегментации, потребности родителей в экспертных рекомендациях и интуитивно понятного интерфейса покупок.

---

## 🚀 Ключевые возможности и архитектура

### 1. Архитектура каталога (Product-Variant Pattern)
Реализована гибкая реляционная структура БД (Django ORM):
* **`Category`**: Логическое деление (Иммунитет, Омега-3, Сон).
* **`Product`**: Базовая карточка товара (описание, хим. состав, противопоказания, возрастные ограничения: 0-3, 3-7, 7+).
* **`ProductVariant`**: Торговые предложения (SKU, форма выпуска, вкус, цена, физический остаток на складе). Позволяет привязывать к одной карточке бесконечное число вариаций без дублирования контента.

### 2. Производительный Backend
* **Защита от N+1:** Упреждающая выборка связей через `select_related('category')` и `prefetch_related('variants')`. Весь каталог со всеми вкусами и изображениями выгружается всего за 2 SQL-запроса.
* **Динамическая фильтрация:** Сквозная фильтрация каталога по форме выпуска, вкусу и возрасту с использованием `.distinct()`.

### 3. Инновационный Auth-Flow (Беспарольный вход)
Современная и безопасная авторизация по OTP-коду (One Time Password), идеальная для целевой аудитории:
* Пользователь вводит Email или телефон (срабатывает автоматический парсинг и валидация регулярными выражениями).
* Сервер генерирует 4-значный код и кладет его в `request.session`.
* При совпадении кода происходит интеллектуальное разветвление: невидимая авторегистрация для новых пользователей или `login()` для существующих. Никаких забытых паролей!

### 4. Медицинский Квиз (Интерактивный подбор)
Пошаговая педиатрическая анкета для родителей, симулирующая первичную консультацию:
* Сбор данных: возраст, главная проблема (рост, иммунитет), любимая форма витаминов.
* Алгоритм анализирует ответы, делает текстовый поиск по БД и генерирует персональную карту рекомендаций с кнопками быстрой покупки.

### 5. Транзакционность и логика заказов
* **Безопасные транзакции:** Использование `transaction.atomic` при оформлении заказа. Историческая стоимость фиксируется, корзина очищается, а остатки на складе (`stock`) синхронно списываются.
* **Повторить заказ:** Кнопка в Личном кабинете для цикличных покупок (витамины пьют курсами). Бэкенд сверяет товары из старого чека с актуальными остатками и переносит их в новую корзину в один клик.

### 6. Живой Frontend (Vanilla JS)
* **Динамические карточки:** При переключении вкуса в `<select>` скрипт мгновенно обновляет цену, артикул и изображение через HTML5 `data-*` атрибуты без перезагрузки страницы.
* **Контроль дефицита:** Если товара на складе < 5 шт., выводится предупреждение. При `stock == 0` кнопка блокируется, срабатывает бэкенд-защита от овербукинга.
* **Адаптивный UI/UX:** Пастельный медицинский дизайн, бесконечные баннеры, адаптивный navbar с «бургером», кастомная 404-страница и обводка логотипа (`-webkit-text-stroke`).

## 🛠 Технологический стек
* **Backend:** Python 3.13, Django 6.0
* **Database:** PostgreSQL (перенос с SQLite)
* **Frontend:** HTML5, CSS3, Vanilla JavaScript, элементы Bootstrap
* **Testing:** Django `TestCase` (Unit-тестирование корзины, авторизации, лимитов склада)

## 💻 Установка и запуск локально

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/vitabears.git
   cd vitabears
   ```

2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Windows: venv\Scripts\activate
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Настройте переменные окружения для подключения к PostgreSQL (в файле `.env`).

5. Примените миграции:
   ```bash
   python manage.py migrate
   ```

6. Запустите сервер разработки:
   ```bash
   python manage.py runserver
   ```
